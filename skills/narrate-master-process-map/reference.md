## When to Use This Skill

Activate when user provides:
- **Multi-page swimlane diagram** (typically 8-15+ pages)
- **Strategic framework** document (usually Level 1.0 processes)
- Document positioned as **"Kerangka Acuan" or "Panduan Navigasi"** in Bab 1
- Heavy **cross-referencing** to other processes (2.0, 3.0, 4.0... and Kantor Pusat)
- Request to create **"panduan navigasi"** or **"penjelasan kerangka proses"**

**Typical characteristics:**
- Multiple sub-processes (e.g., 1.1, 1.2, 1.3, 1.4)
- Strategic stakeholders (Kantor Pusat, leadership level)
- Strategic documents (RJPU, RKAU, Visi & Misi)
- Annual/multi-year cycles

**DON'T use for:**
- Single-page operational process diagrams (use `business-process-narrative` instead)
- Tactical execution procedures
- Level 3-4 detail processes

## Input Expected

User will provide:
- **PDF file path**: Master process map diagram
- **Optional**: Bab number, process number (e.g., "Bab 1, Proses 1.0")
- **Optional**: Target file path for output

## Output Format: Layered Navigation Guide

Create a **layered document** with three integrated approaches:

### Layer 1: Strategic Narrative (Storytelling)
- Executive summary dengan narrative arc
- Big picture flow dan strategic importance
- 2-4 paragraphs opening

### Layer 2: Navigation Tools (Visual + Interpretative)
- Simplified flowchart or process matrix
- Sub-process summaries dengan interkoneksi
- Key control points highlighted
- Role-based navigation guide

### Layer 3: Detailed Narrative (Storytelling + Interpretative)
- Full narrative per sub-process
- Detailed control points dengan checklists
- Links ke SOP detail dan formulir

### Layer 4: Reference
- Link ke original swimlane PDF

## Workflow: Step-by-Step

### Phase 1: Analysis (Read Multi-Page Diagram)

**Step 1: Read entire PDF**
- This is likely 8-15 pages
- Each page typically covers one sub-process
- Understand overall structure first

**Step 2: Identify framework structure**
- Main process number (e.g., 1.0)
- Sub-processes (e.g., 1.1, 1.2, 1.3, 1.4)
- How pages are organized (by sub-process usually)

**Step 3: Extract header information (from page 1)**
- Process title
- Strategic scope
- Stakeholders listed

**Step 4: Map all sub-processes across pages**
Create inventory:
```
Page 1-3: Sub-proses 1.1 (Nama)
Page 4-5: Sub-proses 1.2 (Nama)
Page 6: Sub-proses 1.2.2 (detail)
Page 7: Sub-proses 1.2.3 (detail)
Page 8: Sub-proses 1.3 (Nama)
Page 9-12: Sub-proses 1.4 (Nama)
```

**Step 5: Identify cross-references**
- Kantor Pusat processes (external dependencies)
- Downstream processes (2.0, 3.0, 4.0...)
- Internal connectors (A, B, C... Z between pages)

**Step 6: Identify strategic decision points**
- "Disetujui?" → strategic approval gates
- "Perlu restrukturisasi?" → organizational change decisions
- Not operational decisions like operational diagrams

**Step 7: Map stakeholder ecosystem**
- PT PLN (Persero) Kantor Pusat
- Pusdiklat leadership
- Bidang-bidang (departments)
- BPOs (Business Process Owners)
- Unit Pelaksana

**Step 8: Identify key inputs and outputs**
- Strategic documents IN (Visi Misi, RJPP, RKAP)
- Strategic artifacts OUT (RJPU, RKAU, Kontrak Manajemen)

### Phase 2: Create Layer 1 - Strategic Narrative

**Step 9: Write Gambaran Umum (2-4 paragraphs)**

**Paragraph 1**: What is this framework?
```
Proses Bisnis [nomor] [nama] merupakan kerangka kerja strategis yang
menjadi fondasi bagi seluruh operasional [organisasi]. Framework ini
menggambarkan bagaimana [organisasi] menyelaraskan visi dengan
strategi korporat, menerjemahkan arahan strategis menjadi rencana
jangka panjang dan tahunan, serta memastikan kepatuhan terhadap
regulasi yang berlaku.
```

**Paragraph 2**: Why important?
- Strategic alignment value
- Impact pada operational processes
- Position sebagai "payung"

**Paragraph 3**: Structure overview
- List sub-processes dengan brief function
- Sequential/parallel relationships
- Temporal scope (annual cycle, multi-year)

**Paragraph 4**: Key stakeholders
- Who's involved at strategic level
- Nature of interactions

### Phase 3: Create Layer 2 - Navigation Tools

**Step 10: Create Peta Proses Utama (Table)**

Simple reference table:

| Sub-Proses | Nama | Fungsi Utama | Output Kunci | Connects To |
|------------|------|--------------|--------------|-------------|
| [X.1] | [Nama] | [1-line purpose] | [Main deliverable] | [Downstream] |
| [X.2] | [Nama] | [1-line purpose] | [Main deliverable] | [Downstream] |
| [X.3] | [Nama] | [1-line purpose] | [Main deliverable] | [Downstream] |

**Step 11: Create simplified flowchart (text-based)**

High-level flow showing:
- Start
- Major sub-processes
- Key decision points
- Main outputs
- Connection to operational processes

Use simple ASCII art or markdown structure.

**Step 12: Write Panduan Navigasi untuk Peran**

For common roles, specify:
- Which sub-processes are relevant
- What their responsibilities are
- Where to find details

Template:
```markdown
**Jika Anda adalah [Role]**
- **Fokus utama**: Sub-proses [X.Y]
- **Peran kunci**: [Responsibilities]
- **Lihat detail**: [Specific sections]
```

### Phase 4: Create Layer 3 - Detailed Narrative

**Step 13: For each sub-process, create full section**

Use consistent template:

```markdown
### [X.Y] [Nama Sub-Proses]

**Tujuan Strategis:**
[1-2 kalimat explaining why]

**Narrative Flow:**
[2-4 paragraphs dengan storytelling approach]
- Use transition words
- Show cause-effect
- Explain interconnections naturally
- Humanize (Bidang X melakukan..., Tim Y mengkoordinir...)

**Pihak yang Terlibat:**
- **[Stakeholder 1]**: [Role]
- **[Stakeholder 2]**: [Role]

**Input Utama:**
- [Input 1] (dari: [source])
- [Input 2]

**Output/Deliverable:**
- [Output 1] (digunakan oleh: [destination])
- [Output 2]

**Interkoneksi:**
- Menerima input dari: [sources]
- Memberikan output ke: [destinations]
- Dependency: [what must happen first]

**Lihat Detail Operasional:**
- Bab [X]: [SOP name]
- Formulir: [Form codes]
```

**Step 14: Identify and document control points**

For major decision points and gates:

```markdown
## Titik Kontrol Kritis: [Nama Control Point]

**Lokasi**: Sub-proses [X.Y.Z]

**Deskripsi:**
[Explanation dari diagram - what's being decided/controlled]

**Mengapa Kritis:**
[Impact of failure, consequences]

**Penanggung Jawab:**
[Who has authority]

**Checklist Kepatuhan:**
- [ ] [Requirement 1]
- [ ] [Requirement 2]

**Escalation:**
[What to do if issues arise]
```

**Step 15: Create Alur Strategis dan Timeline**

Narrate the annual/multi-year cycle:

```markdown
### Siklus Tahunan [Process Name]

**Kuartal 1:**
- Sub-proses [X.Y]: [Activities]

**Kuartal 2:**
- Sub-proses [X.Y]: [Activities]

[etc...]

**Note:** Timeline bersifat ilustratif, dapat bervariasi sesuai kebijakan.
```

**Step 16: Document Keterkaitan dengan Proses Lain**

Three subsections:

**Upstream (from Kantor Pusat):**
Table showing KP processes that provide input

**Downstream (to Operational):**
Table showing how outputs feed processes 2.0-13.0

**Internal (inter sub-process):**
Flow diagram showing dependencies among sub-processes

### Phase 5: Generalization & Finalization

**Step 17: Generalize unit-specific references**

Same rules as business-process-narrative:
- Remove location names
- Generic role titles
- Applicable to all units

**Step 18: Add Lampiran section**

```markdown
## Lampiran: Swimlane Diagram Lengkap

Untuk keperluan audit, legal compliance, dan reference detail:

**Dokumen**: [filename.pdf]
**Lokasi**: [path]
**Halaman**: [1-12]

Swimlane diagram original berisi detail lengkap dari seluruh aktivitas,
decision points, dan interkoneksi. Diagram tersebut tetap menjadi sumber
referensi authoritative.

Navigation guide ini dimaksudkan untuk memudahkan pemahaman dan
mempercepat navigasi, bukan menggantikan diagram original.
```

**Step 19: Quality check**

Verify:
- ✅ Strategic narrative flows well (not tactical steps)
- ✅ Multi-page content integrated coherently
- ✅ Visual aids simplify, not complicate
- ✅ Control points identified dan explained
- ✅ Interconnections emphasized
- ✅ Role-based navigation provided
- ✅ Formal Bahasa Indonesia throughout
- ✅ Generalized (no unit-specific)
- ✅ Link to original PDF included

**Step 20: Save file**

Determine filename:
- If user provides notation: `0X-ABC-X.0-FRAMEWORK-Nama.md`
- Otherwise: `Framework-[Process-Name]-Navigation-Guide.md`

## Key Principles

### Strategic Not Tactical

❌ DON'T:
```
1. Petugas melakukan analisis PESTLE dengan langkah:
   a. Buka aplikasi
   b. Input data
   [detailed steps...]
```

✅ DO:
```
Bidang Perencanaan melakukan analisis komprehensif menggunakan framework
PESTLE untuk memahami faktor eksternal. Hasil analisis ini, dikombinasikan
dengan analisis internal (SWOT, IFE-EFE), menjadi dasar perumusan visi
strategis yang aligned dengan Kantor Pusat.
```

### Emphasize Interconnections

This is the MOST IMPORTANT aspect of master process map.

Always explain:
- Where inputs come from
- Where outputs go to
- How sub-processes depend on each other
- Strategic dependencies on Kantor Pusat

### Multi-Audience Layering

- **Executives**: Read Layer 1 (summary)
- **Managers**: Use Layer 2 (navigation tools)
- **Staff**: Reference Layer 3 (detail)
- **Auditors**: Check Layer 4 (original diagram)

### Simplify Visual Complexity

- Use tables instead of reproducing swimlane
- Create high-level flowchart, not detailed flowchart
- Focus on major flows, not every box
- Visual aids should be scannable

### Storytelling for Coherence

- Create narrative arc for each sub-process
- Use transitions to link sub-processes
- Tell the "strategic planning journey"
- Humanize the process (actors doing things)

## Language Requirement

**CRITICAL**: All text in **FORMAL BAHASA INDONESIA**

Same standards as business-process-narrative:
- Third person, professional tone
- Complete sentences
- Italicize foreign terms
- No informal language

## Conservative Approach

### DO:
✅ Extract visible information from diagram
✅ Create strategic narrative explaining framework
✅ Simplify complex visual into navigable structure
✅ Identify control points shown in diagram
✅ Explain interconnections

### DON'T:
❌ Add strategic initiatives not shown
❌ Invent decision criteria not in diagram
❌ Hallucinate references or policies
❌ Change strategic framework logic
❌ Add tactical details (that's for operational SOPs)

### When Uncertain:
- Control point checklist items: Be generic, don't invent specifics
- Timeline: Mark as "ilustratif, dapat bervariasi"
- KP processes: Only mention if clearly referenced in diagram
